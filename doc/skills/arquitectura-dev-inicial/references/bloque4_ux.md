# Bloque 4: Diseño y Experiencia de Usuario (UX/UI)

**Propósito:** Conocer las expectativas de diseño, dispositivos objetivo y accesibilidad.

Preguntas:

1. **¿Tienes algún diseño previo o referencia visual?**
    - Opciones:
        - "Sí, tengo un enlace de Figma / imagen / papel."
        - "No, prefiero empezar desde cero con wireframes de baja fidelidad."
        - "No, pero tengo una idea clara en mente."  
          _Sugerencia:_ Si no tiene diseño, ofrécele que el agente puede proponer una estructura de pantallas después del cuestionario (sin código).

2. **¿El sistema debe ser responsive (adaptarse a móviles y escritorio)?**
    - Opciones: "Sí, fundamental", "Solo escritorio", "Solo móvil".  
      _Sugerencia:_ Casi todos los proyectos web hoy deben ser responsive al menos en parte.

3. **¿Hay restricciones de accesibilidad?**  
   _Ejemplos:_ "Debe cumplir WCAG 2.1 nivel AA (para personas con discapacidad visual)", "No es necesario".  
   _Si no sabe_, sugerir "nivel básico: contraste suficiente, etiquetas en formularios".

4. **¿El sistema estará en varios idiomas?**
    - Opciones: "Solo español", "Español e inglés", "Multilenguaje".  
      _Si es multilenguaje,_ después se planificará cómo (ej. i18n, archivos de traducción).

**Instrucción:** Registra las respuestas. Si el usuario no tiene claro el diseño, anótalo como "por definir – se retomará en la fase de prototipado".
