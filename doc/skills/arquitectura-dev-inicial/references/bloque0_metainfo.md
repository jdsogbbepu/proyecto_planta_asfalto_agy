# Bloque 0: Meta-información

**Propósito:** Conocer el nombre del proyecto, el objetivo principal, el equipo y el contexto general.

Preguntas (con ejemplos y sugerencias):

1. **¿Cuál es el nombre del proyecto?**  
   _Ejemplo:_ "Sistema de control de producción para Planta Asfalto Norte"  
   _Sugerencia:_ Usa un nombre corto y descriptivo. Si no lo tienes, podemos asignarle uno temporal.

2. **¿Cuál es el objetivo principal en una frase?**  
   _Ejemplo:_ "Registrar la producción diaria de asfalto y generar reportes automáticos para la gerencia"  
   _Sugerencia:_ Piensa en qué valor entregará el sistema al negocio o a los usuarios.

3. **¿Eres el único desarrollador o hay más personas en el equipo?**  
   _Ejemplo:_ "Solo yo", "Somos 3 (frontend, backend, BD)", "Todavía no sé".  
   _Sugerencia:_ Si hay más personas, más adelante preguntaré sobre roles y coordinación. Si es solo tú, podremos avanzar más rápido.

4. **¿Hay algún stakeholder o cliente principal que deba ser consultado durante el desarrollo?**  
   _Ejemplo:_ "El gerente de planta", "El departamento de calidad", "No, solo yo".  
   _Sugerencia:_ Identificar a los tomadores de decisiones ayuda a evitar cambios de último momento.

**Instrucción:** Realiza las preguntas en orden. Si alguna respuesta es vaga, pide un ejemplo concreto. Registra las respuestas tal cual o parafrasea si es extenso.
