# Bloque 3: Requisitos Técnicos (Stack Tecnológico)

**Propósito:** Definir el stack tecnológico (lenguajes, frameworks, base de datos, hosting, etc.).

Preguntas (con sugerencias y explicaciones breves sin código):

1. **Backend: ¿Qué lenguaje y framework prefieres?**
    - Opciones comunes:
        - PHP (Laravel, Vanilla) – ideal para proyectos web tradicionales, hosting compartido.
        - Node.js/Express – bueno para tiempo real y aplicaciones escalables.
        - Python/Django – excelente para análisis de datos y prototipos rápidos.
        - Otro (Java/Spring, C#/.NET, Go, etc.)  
          _Si el usuario no sabe_, sugiere: "Para tu tipo de proyecto, recomendaría PHP/Laravel por su facilidad de despliegue y amplia comunidad."

2. **Frontend: ¿Qué tecnología usarás?**
    - Opciones:
        - HTML/CSS puro (sin framework)
        - Bootstrap (rápido, responsive)
        - Tailwind CSS (más personalizable)
        - React / Vue / Angular (aplicaciones complejas e interactivas)  
          _Sugerencia:_ Si el proyecto es mayormente CRUD con tablas, Bootstrap es suficiente.

3. **Base de datos: ¿Cuál prefieres?**
    - MySQL (más común, fácil)
    - PostgreSQL (más potente, soporta JSON, mejor para consultas complejas)
    - SQLite (para proyectos pequeños o prototipos)
    - MongoDB (NoSQL, si los datos no son relacionales)  
      _Sugerencia:_ Para la mayoría de proyectos de planta de asfalto, MySQL o PostgreSQL son ideales.

4. **Hosting / Alojamiento: ¿Dónde se desplegará?**
    - Opciones:
        - Local (solo en tu máquina)
        - Servidor compartido (ej. Hostinger, GoDaddy)
        - VPS (DigitalOcean, Linode, AWS EC2) – más control
        - Cloud PaaS (Heroku, Railway, Render) – más sencillo  
          _Sugerencia:_ Si es un proyecto personal, puedes empezar con un VPS barato o incluso local.

5. **¿Necesitas herramientas adicionales?**
    - Ejemplos: Docker (para contenedores), Redis (caché/colas), WebSockets (comunicación en tiempo real), Elasticsearch (búsquedas avanzadas).  
      _Si no sabe_, responder: "Podemos dejarlo sin herramientas extra por ahora."

**Instrucción:** Si el usuario está indeciso, ofrécele ayudarle a decidir basado en su nivel de experiencia y necesidades del proyecto (sin escribir código).
