# Bloque 2: Requisitos Funcionales (Módulos y Características)

**Propósito:** Listar las funcionalidades principales que debe tener el sistema.

Preguntas:

1. **¿Cuáles son las funcionalidades principales del sistema?**  
   _Ejemplo (para planta de asfalto):_
    - Registrar producción diaria (toneladas, tipo de mezcla)
    - Control de inventario de materias primas (arena, betún, filler)
    - Generar reportes semanales/mensuales
    - Panel de indicadores (KPIs) para el gerente  
      _Sugerencia:_ Pide al usuario que liste de 3 a 10 ítems. Puedes ayudarle con preguntas como "¿Necesitas algo para gestionar clientes o solo producción?"

2. **¿Hay alguna funcionalidad crítica que sea el "corazón" del sistema?**  
   _Ejemplo:_ "El registro de producción es lo más importante; si falla, el negocio se detiene."  
   _Esto ayudará a priorizar._

3. **¿Se necesita autenticación de usuarios?**
    - Opciones: "No, acceso libre", "Sí, solo un usuario administrador", "Sí, con roles múltiples (admin, operario, supervisor)".  
      _Si la respuesta es sí con roles_, pregunta: "¿Qué permisos tendría cada rol?"

4. **¿El sistema debe integrarse con otras aplicaciones o servicios externos?**  
   _Ejemplos:_ "API de facturación electrónica", "exportar datos a Excel", "enviar notificaciones por correo", "conectar a una báscula industrial".  
   _Sugerencia:_ Si no lo sabe, marcar como "sin integraciones por ahora".

**Instrucción:** Anota las funcionalidades en formato de lista en el archivo de respuestas. Pregunta si hay algo que haya olvidado.
