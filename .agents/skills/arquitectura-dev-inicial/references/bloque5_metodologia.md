# Bloque 5: Metodología de Trabajo y Entregables

**Propósito:** Definir cómo se va a desarrollar el proyecto, qué entregables se esperan y plazos.

Preguntas:

1. **¿Qué metodología de desarrollo prefieres?**
    - Opciones:
        - **Ágil con sprints** (entregas cada 1-2 semanas, flexibilidad)
        - **Cascada** (planificar todo al inicio, cambios son costosos)
        - **A demanda** (el agente propone y tú decides)  
          _Sugerencia:_ Para proyectos personales, "a demanda" o "ágil ligero" suele funcionar mejor.

2. **¿Qué entregables quieres además del código funcionando?**  
   _Ejemplos:_
    - Diagrama de base de datos (DER)
    - Documentación de API (Swagger/OpenAPI)
    - Manual de usuario
    - Diagrama de arquitectura (Mermaid)
    - Reporte de pruebas  
      _Puede seleccionar varios._

3. **¿Hay alguna fecha límite (aproximada)?**  
   _Ejemplo:_ "En un mes me gustaría tener un MVP (mínimo producto viable)", "Sin fecha apretada".  
   _Esto ayuda a priorizar._

4. **¿Quieres que el agente proponga la arquitectura detallada, o tú ya tienes una en mente?**
    - Opciones: "Que proponga", "Yo la definiré", "Trabajaremos juntos".

**Instrucción:** Asegúrate de que el usuario entienda que los entregables pueden ajustarse después del análisis.
